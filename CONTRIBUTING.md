# Contributing to Vietnam Assistive Technology Lab

Thank you for your interest in contributing to this project! This deployment kit helps bring accessible technology education to blind children in Vietnam.

## How to Contribute

### 1. **Test the Deployment Process**
- Run the scripts on a test Windows 11 machine
- Report any bugs or issues in GitHub Issues
- Suggest improvements to the automation

### 2. **Improve Documentation**
- Translate materials to Vietnamese (or improve existing translations)
- Add troubleshooting tips based on real experiences
- Create video tutorials or screenshots

### 3. **Enhance Scripts**
- Improve error handling in PowerShell scripts
- Add new features (e.g., parallel installation)
- Optimize installation process

### 4. **Add Training Materials**
- Contribute Vietnamese-language tutorials
- Create accessible learning content
- Develop practice exercises for students

## Getting Started

1. Fork this repository
2. Create a feature branch: `git checkout -b feature/your-feature-name`
3. Make your changes
4. Test thoroughly on Windows 11
5. Commit with clear messages: `git commit -m "Add: Vietnamese troubleshooting guide"`
6. Push to your fork: `git push origin feature/your-feature-name`
7. Open a Pull Request

## Code Style

### PowerShell Scripts
- Use verbose variable names
- Add comments for complex logic
- Include error handling with Try/Catch
- Log all operations to files
- Test in both PowerShell 5.1 and 7.x

### Documentation
- Use clear, simple language (for non-technical users)
- Provide both English and Vietnamese versions
- Include screenshots where helpful
- Format with Markdown for readability

## Testing Checklist

Before submitting a PR, verify:

- [ ] Scripts run without errors on clean Windows 11
- [ ] All software installs successfully offline
- [ ] NVDA launches and speaks in Vietnamese
- [ ] Documentation is clear and accurate
- [ ] No personal information or API keys in commits

## Code of Conduct

This project serves vulnerable children. All contributions must:
- Be respectful and inclusive
- Prioritize accessibility and usability
- Consider the limited resources of the target environment
- Support the educational mission
